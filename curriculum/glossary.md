# Glossary

Plain-English translations of common dev terms. New terms get added here as each chapter introduces them.

- **Repo** — short for "repository." A project folder that has version history built in. Like a regular folder but it remembers every change you've ever made and can roll any of them back.
- **GitHub** — a website that hosts repos on the internet. Think of it like Google Drive but for code projects, with extra features for tracking changes and collaborating.
- **Clone** — making a copy of a GitHub repo onto your computer. Your laptop and GitHub now each have a copy, and they stay in sync when you tell them to.
- **Terminal** — a window where you type commands instead of clicking buttons. Looks intimidating; isn't. We barely use it.
- **Claude Code** — an AI coding assistant that runs on your computer and can build software by talking to it in English.
- **Branch** — a testing lane. A safe copy of your code where you can try changes without affecting the main version.
- **Commit** — a save point. A snapshot of your project at a specific moment.
- **PR (Pull Request)** — a request to merge a branch into main. It's the review step before changes become official.
- **Main** — the primary branch. The "real" version of your project. Everything eventually merges back here.
- **Merge** — combining a branch's changes into another branch (usually main).
- **Vite** — a tool that takes your code, compiles it, and serves it at a local web address so you can see it in a browser.
- **React** — a library for building user interfaces using reusable pieces called "components."
- **TypeScript** — JavaScript with type-checking. Catches mistakes before they become bugs.
- **Component** — a reusable piece of UI. A button is a component. A navigation bar is a component.
- **Dev server** — a local web server running on your computer that shows you your app while you build.
